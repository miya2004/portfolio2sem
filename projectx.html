Canvas setup

The game initializes a canvas element and its 2D rendering context (c). The canvas dimensions are set to 1024x576, and a gravity constant is defined to simulate falling,Click here to see my git,Click here to see my game

Player

I made a player class where all the properties are there, a draw method to put the player as an image and a update method handles movement, collision detection, and gravity effects

Platforms

Platforms are represented by the Platform class, which takes a position and an image as parameters. Platforms are drawn using the draw method, and their positions are updated dynamically as the player moves

Enemies

The Enemy class defines enemy objects with properties such as position, size, speed, and movement direction. Enemies move horizontally, reversing direction when they hit the edges of the canvas, and can detect collisions with the player

Game Initialization:s

The init function initializes the game by creating a new player, a set of platforms, and enemies, while resetting the scroll offset

Dynamic Content:

Functions like addPlatforms and addEnemies dynamically generate new platforms and enemies as the player progresses

animate function

The animate function is the main game loop, responsible for rendering the background, updating positions of the player, platforms, and enemies, handling collisions, and adding new elements when necessary

keyboard controls

The keydown and keyup event listeners handle player movement. The 'A' key moves left, the 'D' key moves right, and the 'W' key makes the player jump, with the corresponding logic adjusting the player's velocity and direction
